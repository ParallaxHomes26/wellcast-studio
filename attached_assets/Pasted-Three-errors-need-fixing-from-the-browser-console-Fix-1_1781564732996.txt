Fix 2 — Profile fetch permission denied (RLS 42501)

The profiles RLS policy is still blocking reads for authenticated users. Run this SQL in Supabase:
sqlDROP POLICY IF EXISTS "Users can view own profile" ON profiles;

CREATE POLICY "Users can view own profile"
ON profiles FOR SELECT
TO authenticated
USING (auth.uid() = id);
The key addition is TO authenticated — this explicitly grants the policy to authenticated users only.
